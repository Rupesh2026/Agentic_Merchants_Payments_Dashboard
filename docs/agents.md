# agents.md — Agentic RAG Design

## Overview

The AI Analyst is a **LangChain agentic RAG system** backed by:
- **pgvector** on PostgreSQL — vector store for merchant summaries and transaction context
- **Claude API** (claude-sonnet-4-6) — reasoning and generation
- **SQL tools** — live queries against Gold layer for fresh KPI data
- **SHAP integration** — ML explainability piped into the agent context

The agent does NOT just do RAG retrieval. It has tools it can call to answer questions,
making it agentic: it decides which tool(s) to use based on the question.

---

## Architecture

```
User question
     │
     ▼
┌────────────────────────────────────────────────────────┐
│  LangChain Agent  (rag/agent.py)                        │
│                                                         │
│  System prompt + conversation history                   │
│       │                                                 │
│       ├─► Tool: vector_search    → pgvector retrieval   │
│       ├─► Tool: sql_query        → Gold layer SQL       │
│       ├─► Tool: get_merchant     → merchant stats       │
│       ├─► Tool: explain_fraud    → SHAP values          │
│       └─► Tool: get_kpi          → KPI summary          │
│                                                         │
│  Claude synthesizes tool outputs → final answer         │
└────────────────────────────────────────────────────────┘
     │
     ▼
Natural language answer with citations
```

## RAG Corpus — What Gets Embedded

### 1. Merchant Summaries (primary corpus — 800 documents)
Generated by Claude from `gold.merchant_stats`. One document per merchant.

**Template:**
```
Merchant: {merchant_name}
Category: {category}
Total Volume: ${total_volume:,.2f}
Total Transactions: {total_transactions:,}
Average Ticket: ${avg_ticket:.2f}
Fraud Rate: {fraud_rate:.4%}
Chargeback Count: {chargeback_count}
Approval Rate: {approval_rate:.2%}
Average Risk Score: {risk_score_avg:.1f}/100
Active Period: {first_seen} to {last_seen}

Risk Assessment:
{claude_generated_narrative}
```

**Claude-generated narrative prompt** (called during `rag/embeddings.py`):
```
Given the following merchant statistics, write a 2-3 sentence risk assessment
for a fraud analyst. Focus on: fraud rate vs industry average (0.58%),
volume trends, and any concerning patterns. Be specific and data-driven.
Be concise — this is for a retrieval corpus, not a report.

Stats: {stats_json}
```

### 2. Daily Reports (30 documents — last 30 days)
Summarized from `gold.kpi_daily`. One document per day.

**Template:**
```
Date: {date}
Gross Volume: ${gross_volume:,.2f} ({volume_delta:+.1%} vs prior day)
Transactions: {total_transactions:,}
Fraud Rate: {fraud_rate:.4%}
Approval Rate: {approval_rate:.2%}
Chargebacks: {chargeback_count}
Top Category: {top_category}
Notable: {anomaly_note}
```

### 3. Fraud Incident Reports (flagged clusters)
Generated when a fraud spike is detected in Gold layer.
One document per detected cluster event.

**Template:**
```
Fraud Incident: {incident_date}
Pattern: {pattern_type}  (e.g. "Velocity spike", "Category anomaly", "Geographic cluster")
Merchants Affected: {merchant_list}
Transaction Count: {tx_count}
Total Amount: ${total_amount:,.2f}
Risk Scores: avg={avg_risk}, max={max_risk}
Time Window: {start_time} to {end_time}
```

## Agent Tools

### Tool 1: `vector_search`
```python
def vector_search(query: str, doc_type: str = None, top_k: int = 5) -> list[dict]:
    """
    Semantic search over gold.embeddings using pgvector cosine similarity.
    
    Args:
        query: Natural language query to embed and search
        doc_type: Filter by 'merchant_summary', 'daily_report', 'fraud_incident'
        top_k: Number of results to return (default 5)
    
    Returns:
        List of {content, doc_id, doc_type, similarity_score, metadata}
    
    When to use:
        - "Tell me about merchant X" → search merchant summaries
        - "What happened on June 3rd?" → search daily reports
        - "Any fraud clusters recently?" → search fraud incidents
    """
```

### Tool 2: `sql_query`
```python
def sql_query(query: str) -> pd.DataFrame:
    """
    Execute a read-only SQL query against the Gold layer.
    Only SELECT statements allowed. Only gold.* and clean.* schemas.
    
    When to use:
        - "What is today's fraud rate?" → SELECT from gold.kpi_daily
        - "Which merchants have the highest volume?" → SELECT from gold.merchant_stats
        - "Show me transactions over $1000" → SELECT from clean.transactions
    
    IMPORTANT: Never modify data. Only SELECT.
    """
```

### Tool 3: `get_merchant_detail`
```python
def get_merchant_detail(merchant_name: str) -> dict:
    """
    Get full stats for a specific merchant from gold.merchant_stats.
    
    Returns:
        Full merchant record including volume, fraud_rate, risk scores,
        top categories, recent trend (7d vs 30d comparison)
    
    When to use:
        - "What is the risk profile of TravelBook?"
        - "How is Acme Electronics performing?"
    """
```

### Tool 4: `explain_fraud_transaction`
```python
def explain_fraud_transaction(trans_num: str) -> dict:
    """
    Get ML model explanation for a specific transaction.
    Retrieves SHAP values from gold.risk_scores.top_shap_features.
    
    Returns:
        {
          risk_score: int,
          fraud_probability: float,
          top_features: [
            {"feature": "geo_distance_km", "value": 892.3, "shap_impact": 0.41,
             "explanation": "Transaction occurred 892km from cardholder's home city"},
            {"feature": "tx_count_1h",     "value": 8,     "shap_impact": 0.38,
             "explanation": "8 transactions in last hour — unusually high velocity"},
            {"feature": "is_online",        "value": True,  "shap_impact": 0.21,
             "explanation": "Online transaction category carries elevated fraud risk"}
          ]
        }
    
    When to use:
        - "Why was transaction TX123 flagged?"
        - "Explain the fraud score for this transaction"
    """
```

### Tool 5: `get_kpi_summary`
```python
def get_kpi_summary(period: str = "today") -> dict:
    """
    Get KPI summary for a time period.
    period: 'today', 'yesterday', '7d', '30d', 'mtd', 'ytd'
    
    Returns:
        {gross_volume, transactions, fraud_rate, approval_rate,
         chargebacks, avg_ticket, vs_prior_period}
    
    When to use:
        - "How are we performing this month?"
        - "What's today's fraud rate?"
        - "Give me a weekly summary"
    """
```

## System Prompt

```
You are an expert payments fraud analyst AI assistant for a merchant dashboard.
You have access to real transaction data for 800 merchants across 1.85 million transactions.

Your capabilities:
- Analyze merchant performance (volume, approval rates, trends)
- Investigate fraud patterns (velocity, geographic anomalies, category risk)
- Explain ML model decisions using SHAP feature importance values
- Answer natural language questions about any transaction, merchant, or time period
- Generate risk summaries and actionable recommendations

Your tools:
- vector_search: semantic search over merchant summaries and daily reports
- sql_query: live queries against the Gold data layer
- get_merchant_detail: detailed stats for a specific merchant
- explain_fraud_transaction: ML explanation for a specific transaction
- get_kpi_summary: KPI metrics for any time period

Response style:
- Be specific and data-driven — always cite actual numbers
- Flag anomalies proactively when you see them in retrieved data
- When explaining fraud, reference specific SHAP features in plain English
- Keep responses concise — this is a dashboard, not a report
- If a question needs multiple tools, call them all before synthesizing

Important context:
- Industry average fraud rate: 0.58% (this dataset's baseline)
- Fraud flagging threshold: risk_score > 70
- High-risk categories: travel, entertainment, misc_net (online)
- Low-risk categories: grocery_pos, gas_transport, health_fitness
```

## Example Agent Interactions

### Example 1 — Performance Question
```
User: "Why did revenue drop on June 8th?"

Agent reasoning:
1. Call get_kpi_summary("2026-06-08") → confirms drop, -23% vs prior day
2. Call sql_query("SELECT category, SUM(amt) FROM clean.transactions 
   WHERE DATE(trans_at)='2026-06-08' GROUP BY category ORDER BY 2 DESC")
   → travel and entertainment down most
3. Call vector_search("June 8 fraud incident anomaly", doc_type="daily_report")
   → finds daily report mentioning elevated decline rate

Response: "Revenue on June 8th was $X, down 23% vs June 7th. The primary driver 
was a 41% drop in travel category transactions ($X → $X), likely linked to an 
elevated decline rate of 4.2% on that day (vs your 2.7% average). The fraud_heatmap 
shows a spike at 14:00-16:00 UTC on that day."
```

### Example 2 — Fraud Explanation
```
User: "Why was transaction TX-abc123 flagged?"

Agent reasoning:
1. Call explain_fraud_transaction("TX-abc123") → gets SHAP values
2. Call get_merchant_detail(merchant_from_trans) → context on that merchant

Response: "Transaction TX-abc123 received a risk score of 88/100. The three main 
signals were:
• **Geographic distance** (contributes 41% of score): The transaction occurred 
  892km from the cardholder's registered city — highly unusual.
• **Transaction velocity** (38%): 8 transactions in the prior hour — this customer 
  normally averages 1.2/hour.
• **Online category** (21%): misc_net transactions carry 2.3x the baseline fraud rate.

The merchant (Acme Electronics) has a historical fraud rate of 0.9% — above the 
0.58% average — which added additional context."
```

### Example 3 — Merchant Analysis
```
User: "Which merchants should I be worried about?"

Agent reasoning:
1. Call sql_query("SELECT merchant, fraud_rate, total_volume, risk_score_avg 
   FROM gold.merchant_stats WHERE fraud_rate > 0.01 ORDER BY fraud_rate DESC LIMIT 10")
2. Call vector_search("high risk merchant fraud spike", doc_type="merchant_summary", top_k=3)

Response: "Top 3 merchants to watch:
1. **TravelBook** — fraud rate 1.8% (3x average), avg risk score 74. Seeing velocity 
   spikes on weekend evenings.
2. **FastTech Online** — fraud rate 1.4%, high online transaction share (misc_net). 
   Chargeback count up 40% this month.
3. **LuxuryGoods Co** — fraud rate 1.1%, high avg ticket ($380). Most fraud transactions 
   are card-not-present."
```

## ML ↔ RAG Integration

This is the key architectural differentiator — show this in the interview.

```
                    ┌──────────────────────────────┐
                    │  XGBoost Model               │
                    │  Input: 15 engineered features│
                    │  Output: fraud_probability    │
                    └──────────┬───────────────────┘
                               │
                    ┌──────────▼───────────────────┐
                    │  SHAP Explainer              │
                    │  Input: model + feature vals  │
                    │  Output: top 3 SHAP features │
                    │          + contribution %     │
                    └──────────┬───────────────────┘
                               │ stored in
                               │ gold.risk_scores.top_shap_features (JSONB)
                               │
                    ┌──────────▼───────────────────┐
                    │  RAG Agent Tool:             │
                    │  explain_fraud_transaction() │
                    │                              │
                    │  Retrieves SHAP JSON +       │
                    │  merchant context from       │
                    │  pgvector                    │
                    └──────────┬───────────────────┘
                               │
                    ┌──────────▼───────────────────┐
                    │  Claude API                  │
                    │  Translates SHAP features    │
                    │  into plain English          │
                    │  explanation for analyst     │
                    └──────────────────────────────┘
```

**The integration point**: `gold.risk_scores.top_shap_features` is a JSONB column
that stores the SHAP output as:
```json
[
  {"feature": "geo_distance_km",  "value": 892.3, "shap_value": 0.41},
  {"feature": "tx_count_1h",      "value": 8,     "shap_value": 0.38},
  {"feature": "is_online",        "value": true,  "shap_value": 0.21}
]
```
The RAG agent reads this JSON and Claude translates it into the natural language
explanation. No extra API calls needed — SHAP runs at scoring time in `ml/predict.py`.

## Embeddings Pipeline (`rag/embeddings.py`)

```python
# Step 1: Generate merchant summaries from gold.merchant_stats
# Step 2: For each merchant, call Claude API to generate narrative
# Step 3: Embed the full merchant summary text (stats + narrative)
# Step 4: INSERT into gold.embeddings

# Step 5: Generate daily report text from gold.kpi_daily (last 90 days)
# Step 6: Embed each daily report
# Step 7: INSERT into gold.embeddings

# Step 8: Generate fraud incident reports from anomaly detection
# Step 9: Embed each incident
# Step 10: INSERT into gold.embeddings

# Step 11: Build HNSW index on gold.embeddings
```

**Embedding model**: Use Anthropic embeddings if available, otherwise OpenAI
`text-embedding-3-small` (1536 dimensions, fast, cheap, high quality).

**Total documents to embed**: ~830 (800 merchants + 30 daily reports)
**Estimated cost**: < $0.10 at OpenAI pricing

## Agentic Loop (`rag/agent.py`)

Use `langchain` with `AgentExecutor`:

```python
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_anthropic import ChatAnthropic
from langchain.tools import StructuredTool

llm = ChatAnthropic(model="claude-sonnet-4-6", max_tokens=1000)

tools = [
    StructuredTool.from_function(vector_search),
    StructuredTool.from_function(sql_query),
    StructuredTool.from_function(get_merchant_detail),
    StructuredTool.from_function(explain_fraud_transaction),
    StructuredTool.from_function(get_kpi_summary),
]

agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True, max_iterations=5)
```

**Conversation memory**: Store last 10 turns in Streamlit session state,
pass as `chat_history` to agent on each invocation.

## Error Handling

```python
# In agent tools — always wrap DB calls
try:
    result = db.execute(query)
except Exception as e:
    return {"error": str(e), "fallback": "Unable to retrieve data. Try rephrasing."}

# In embeddings — handle API rate limits
import time
def embed_with_retry(texts, max_retries=3):
    for attempt in range(max_retries):
        try:
            return openai_client.embeddings.create(input=texts, model=EMBEDDING_MODEL)
        except RateLimitError:
            time.sleep(2 ** attempt)
    raise Exception("Embedding API unavailable after retries")
```
